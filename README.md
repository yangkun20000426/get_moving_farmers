# 游戏化运动激励工具 - 产品设计文档

## 1. 产品概述

### 1.1 产品定位
**FitFarm（健身农场）** - 将现实运动与虚拟农场经营结合，通过运动打卡获取金币和资源，逐步解锁作物、建筑、宠物，形成强烈的期待感驱动，让用户像沉迷游戏一样沉迷健身。

### 1.2 目标用户
- 大学生群体
- 运动类型：综合运动（跑步、健身、球类等）
- 痛点：缺乏持续动力，难以坚持锻炼计划

### 1.3 核心价值
- **期待感驱动**：永远有未解锁的内容吸引用户
- **正向反馈**：运动直接转化为游戏收益
- **社交绑定**：好友互动增加运动粘性
- **循环激励**：游戏产出再投入运动，形成正向循环

---

## 2. 游戏世界观

### 2.1 核心概念
用户经营一个虚拟农场，通过运动获取金币和资源，种植作物、建造建筑、喂养宠物，不断扩大农场规模。

### 2.2 游戏货币体系
| 货币 | 获取方式 | 用途 |
|------|---------|------|
| 金币 | 运动时长、任务奖励、出售作物 | 购买种子、建造建筑、升级设施 |
| 钻石 | 成就解锁、连续打卡奖励、充值 | 加速成长、购买稀有物品 |
| 体力 | 运动消耗卡路里转化 | 进行高强度运动 |

---

## 3. 运动与游戏挂钩机制

### 3.1 运动数据产出表

| 运动类型 | 运动时长产出 | 消耗卡路里产出 | 单次打卡产出 |
|---------|------------|--------------|------------|
| 跑步 | 10金币/10分钟 | 1金币/10千卡 | 收获1次 |
| 健身房 | 15金币/10分钟 | 1.5金币/10千卡 | 收获1次 |
| 球类运动 | 12金币/10分钟 | 1.2金币/10千卡 | 收获1次 |
| 骑行 | 10金币/10分钟 | 0.8金币/10千卡 | 收获1次 |
| 其他运动 | 8金币/10分钟 | 1金币/10千卡 | 收获1次 |

### 3.2 连续打卡奖励
- **连续7天** → 解锁新作物种子
- **连续14天** → 解锁新建筑
- **连续30天** → 稀有宠物/装饰
- **连续100天** → 传奇称号+专属雕像

### 3.3 运动等级与段位
- 青铜运动员 → 白银运动员 → 黄金运动员 → 铂金运动员 → 钻石运动员
- 每次升级解锁更高价值种子和建筑

## 5. 作物用途系统

- 作物可在市场出售

## 8. UI/UX 设计建议

### 8.1 色彩体系
- 主色：植物绿色 (#4CAF50)
- 辅助色：阳光黄色 (#FFC107)
- 背景色：草地绿色 (#E8F5E9)
- 强调色：活力橙色 (#FF9800)

### 8.2 核心页面
1. **首页** - 农场全景 + 运动记录入口
2. **运动页** - 运动记录 + 实时收益显示
3. **商店页** - 种子/建筑/道具购买
4. **仓库页** - 作物/道具存储
5. **社交页** - 好友/排行榜/偷菜
6. **成就页** - 成就进度/任务/奖励

### 8.3 交互设计
- 运动时有收益飘字动画
- 收获时有弹跳动画
- 解锁新物品有闪亮特效
- 连续打卡有火焰动画

## 9. 核心激励机制

### 9.1 期待感链条
1. 运动 → 获得金币/资源
2. 金币 → 购买种子/升级建筑
3. 作物 → 收获/出售/合成
4. 资源 → 解锁更多内容
5. 更多内容 → 更大的期待

### 9.2 上瘾设计
- **即时反馈**：运动时有收益飘字
- **进度可视化**：进度条/等级/成就
- **社交比较**：排行榜/偷菜互动
- **损失厌恶**：体力会过期、作物会枯萎
- **惊喜元素**：随机订单、幸运掉落

## 附录：GitHub Pages 与本地不一致时

仓库若使用 **Settings → Pages → Deploy from branch → `/docs`**，线上站点来自 **`docs/` 目录里的静态文件**，不是每次 push 源码自动重新构建。

- 本地 `npm run dev` 始终是最新源码。
- 更新线上页面：在项目根执行 **`npm run gh-pages`**（会先 `vite build`，再把 `dist/` 整份同步到 `docs/`），然后 **`git add docs` 并 push**。  
- 若仍看到旧版：浏览器 **强制刷新**（Ctrl+F5）或清空该站点缓存；CDN 偶尔有几分钟延迟。
